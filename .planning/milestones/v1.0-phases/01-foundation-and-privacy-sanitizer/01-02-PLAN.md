<feature>
  <name>Privacy Sanitizer</name>
  <files>src/sanitizer.py, tests/test_sanitizer.py</files>
  <behavior>
    - sanitize(RawMessage with HTML body, config) -> CleanMessage with plain text, no HTML tags (PRIV-01)
    - All img tags removed from output, including tracking pixels (PRIV-02)
    - URLs in output have utm_*, mc_eid, fbclid, gclid, and all TRACKING_PARAMS stripped; non-tracking params preserved (PRIV-03)
    - config.user_email never appears in clean_text or sender_domain (PRIV-04)
    - config.user_name never appears in clean_text or subject (PRIV-04)
    - Extra regex patterns from config.extra_patterns match and replace with [REDACTED] (PRIV-05)
    - sender_domain is domain-only, no @ sign (PRIV-06)
    - clean_text length <= config.max_body_chars (PRIV-07)
    - Edge: body_html=None falls back to body_text
    - Edge: both body_html and body_text None produces empty clean_text
    - Edge: subject containing user_name gets PII redacted
  </behavior>
  <implementation>
Implement src/sanitizer.py following the research patterns exactly:

1. **TRACKING_PARAMS** — frozenset of all known tracking URL parameters (from research Pattern 3). Include all params listed there.

2. **URL_PATTERN** — `re.compile(r'https?://[^\s<>"\']+')` for finding URLs in plain text.

3. **_html_to_text(html: str) -> str** — BeautifulSoup4 with html.parser:
   - `soup.find_all(["script", "style", "head", "meta", "link"])` -> decompose each
   - `soup.find_all("img")` -> decompose each (removes ALL images, conservative approach per research)
   - `soup.get_text(separator="\n", strip=True)`
   - Collapse 3+ newlines to double newline with `re.sub(r"\n{3,}", "\n\n", text)`
   - Return stripped text

4. **_strip_tracking_params(url: str) -> str** — urllib.parse round-trip:
   - Parse URL, filter query params against TRACKING_PARAMS (case-insensitive) and any `utm_*` prefix
   - Reconstruct URL. On any exception, return original URL unchanged.

5. **_strip_tracking_urls(text: str) -> str** — Find all URLs via URL_PATTERN, replace each with _strip_tracking_params result.

6. **_extract_domain(sender: str) -> str** — email.utils.parseaddr then split on @ to get domain. Lowercase. Fallback to "unknown".

7. **_build_redaction_patterns(config: SanitizerConfig) -> list[re.Pattern]** — Build compiled patterns for user_email (re.escape, IGNORECASE), user_name (word-boundary, only if >= 3 chars), and each extra_pattern.

8. **_redact_pii(text: str, config: SanitizerConfig) -> str** — Apply all patterns, replacing matches with "[REDACTED]".

9. **sanitize(raw: RawMessage, config: SanitizerConfig) -> CleanMessage** — Orchestrates the pipeline:
   - Extract text: _html_to_text if body_html, else body_text, else ""
   - Strip tracking URLs
   - Redact PII
   - Truncate to max_body_chars
   - Redact PII from subject too (per research Open Question 3)
   - Extract sender domain
   - Return CleanMessage

**Order matters:** HTML-to-text FIRST, then URL stripping, then PII redaction, then truncation. Per research anti-pattern: never run redaction on raw HTML.

After implementation, if any test needs minor adjustment to match the actual behavior (e.g., whitespace differences), update the test to be correct while still asserting the security invariant. Never weaken a security assertion to make a test pass.
  </implementation>
