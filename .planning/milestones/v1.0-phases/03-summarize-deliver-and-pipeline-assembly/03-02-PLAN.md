<feature>
  <name>TDD: Delivery module — SMTP email + markdown archive + markdown-to-HTML converter</name>
  <files>src/deliver.py, tests/test_deliver.py</files>
  <behavior>
    DLVR-01 — SMTP email delivery:
    - send_digest_email(markdown_text, html_text, config) sends via smtplib.SMTP
    - Uses STARTTLS with ssl.SSLContext(PROTOCOL_TLS_CLIENT, check_hostname=False, CERT_NONE)
    - Calls starttls() BEFORE login() (security requirement)
    - Constructs MIMEMultipart('alternative') with text/plain + text/html parts
    - Subject line: "Daily Digest — YYYY-MM-DD"
    - From: config['imap_username'], To: config['digest_recipient']

    DLVR-02 — Markdown archive:
    - save_archive(digest_md, config) -> Path
    - Saves to {output_dir}/digest-YYYY-MM-DD.md
    - Creates output_dir with mkdir(parents=True, exist_ok=True)
    - Returns the Path of the saved file

    Markdown-to-HTML converter:
    - markdown_to_html(md: str) -> str
    - Converts ## headers to h2, ### to h3
    - Converts - and * bullets to ul/li
    - Converts **bold** to strong
    - Wraps in html/body envelope with basic styling
    - Handles blank lines as paragraph breaks
  </behavior>
  <implementation>
    RED phase: Create tests/test_deliver.py with these tests:
    - test_send_email_calls_starttls: mock smtplib.SMTP, verify starttls called
    - test_send_email_login_after_starttls: verify login called after starttls (use call_args_list order)
    - test_send_email_sends_message: verify send_message called with a MIMEMultipart
    - test_email_has_html_part: verify the sent message contains text/html content type
    - test_email_subject_contains_date: verify subject matches "Daily Digest — YYYY-MM-DD"
    - test_save_archive_creates_file: use tmp_path, verify file created with correct content
    - test_save_archive_filename_format: verify filename matches digest-YYYY-MM-DD.md
    - test_save_archive_creates_directory: use tmp_path with non-existent subdir, verify mkdir
    - test_save_archive_returns_path: verify Path object returned
    - test_markdown_to_html_headers: pass "## Title", verify h2 in output
    - test_markdown_to_html_bullets: pass "- item", verify ul/li in output
    - test_markdown_to_html_bold: pass "**bold**", verify strong in output
    - test_markdown_to_html_wraps_in_html: verify output starts with html tag

    GREEN phase: Create src/deliver.py implementing:
    - send_digest_email(markdown_text: str, html_text: str, config: dict) -> None
    - save_archive(digest_md: str, config: dict) -> Path
    - markdown_to_html(md: str) -> str (plus _apply_inline helper)

    SSL context: ssl.SSLContext(ssl.PROTOCOL_TLS_CLIENT) with check_hostname=False
    and verify_mode=ssl.CERT_NONE — same pattern as IMAP fetch (localhost Bridge only).

    Use smtplib.SMTP as context manager. Call starttls(context=ctx) then login().
    Use email.mime.multipart.MIMEMultipart('alternative') + MIMEText for parts.

    Anti-patterns to avoid:
    - No SMTP_SSL (Bridge uses STARTTLS, not SSL-on-connect)
    - No login() before starttls() (sends credentials in cleartext)
    - No directory creation at module import time (only inside save_archive)
    - No sys.exit() in library code — raise exceptions
  </implementation>
</feature>
