    - test_time_window_filter: Mock two messages — one with Date header within 24h, one 48h old. Verify only the recent one is returned (FETCH-04)
    - test_empty_folder_returns_empty_list: Mock SEARCH returning empty data. Verify fetch_messages returns [] (FETCH-04)
    - test_html_part_preferred: Build a multipart/alternative MIME message with both text/html and text/plain parts. Call _extract_body() directly. Verify html_body contains the HTML content and text_body contains the plain text (FETCH-05)

    - test_connection_not_left_open_on_error: Mock IMAP4 to raise during uid('SEARCH'). Verify the context manager's __exit__ was called (FETCH-01)

    GREEN phase — implement src/fetch.py:
    - _build_ssl_context() — returns SSLContext with CERT_NONE for Bridge self-signed cert
    - _extract_body(msg) — walk() MIME tree, prefer text/html, fallback text/plain, charset decode with fallback
    - _is_within_window(msg, since) — parse Date header, compare to UTC cutoff
    - _sender_matches(msg, allowed) — case-insensitive substring match on From header
    - fetch_messages(config) — connect IMAP4 + STARTTLS, login, select folder, uid SEARCH SINCE, uid FETCH RFC822, parse MIME, filter by time + sender, build RawMessage list
