<feature>
  <name>TDD: Summarization module — Claude CLI subprocess + prompt loading + newsletter concatenation</name>
  <files>src/summarize.py, prompts/summarize.txt, tests/test_summarize.py</files>
  <behavior>
    SUMM-01 — Claude CLI call:
    - call_claude(prompt_file, newsletter_text, config) calls subprocess.run with input=newsletter_text
    - The command includes claude -p "<prompt_content>" --output-format text
    - FileNotFoundError raised when claude binary is not found
    - RuntimeError raised when claude exits non-zero (includes stderr in message)
    - RuntimeError raised when claude returns empty stdout

    SUMM-02/03/04 — Prompt content:
    - prompts/summarize.txt instructs grouping by theme/topic, not per-newsletter
    - Prompt requests key trends, notable announcements, and actionable insights
    - Prompt instructs to flag contradictions between sources

    SUMM-05 — Word count target:
    - Prompt contains word count target (default ~500 words)
    - call_claude accepts optional word_target parameter to customize

    SUMM-06 — Source listing:
    - Prompt instructs Claude to list sources (domain + subject) at end

    SUMM-07 — External prompt file:
    - Prompt loaded from prompts/summarize.txt via Path.read_text()
    - FileNotFoundError if prompt file missing

    Newsletter concatenation:
    - format_newsletter_input(messages: list[CleanMessage]) -> str
    - Each message prefixed with "Source: {domain} | Subject: {subject} | Date: {date}"
    - Messages separated by "---" dividers
    - Empty list returns empty string
  </behavior>
  <implementation>
    RED phase: Create tests/test_summarize.py with these tests:
    - test_call_claude_success: mock subprocess.run returning valid output, verify input= kwarg and -p flag
    - test_call_claude_binary_not_found: mock subprocess.run raising FileNotFoundError
    - test_call_claude_nonzero_exit: mock subprocess.run returning returncode=1, verify RuntimeError
    - test_call_claude_empty_output: mock subprocess.run returning empty stdout, verify RuntimeError
    - test_prompt_contains_theme_instruction: read prompts/summarize.txt, assert "topic" or "theme" present
    - test_prompt_contains_contradiction_instruction: assert "contradict" present in prompt
    - test_prompt_contains_word_target: assert "500" or "word" present in prompt
    - test_prompt_contains_sources_instruction: assert "source" present in prompt
    - test_prompt_loaded_from_file: call call_claude with a tmp_path prompt file, verify it reads from that path
    - test_format_newsletter_input: pass 2 CleanMessages, verify output contains source headers and dividers
    - test_format_newsletter_input_empty: pass empty list, verify empty string returned

    Create prompts/summarize.txt with the prompt content from Pattern 7 in 03-RESEARCH.md.

    GREEN phase: Create src/summarize.py implementing:
    - format_newsletter_input(messages: list[CleanMessage]) -> str
    - call_claude(prompt_file: str, newsletter_text: str, config: dict) -> str
    Use subprocess.run(input=..., capture_output=True, text=True). Never use Popen.
    Resolve prompt_file path with Path() for flexibility (callers can pass absolute or relative).
    Do NOT call sys.exit() — raise exceptions for the orchestrator to catch.
    Do NOT call load_dotenv() or load_config() — receive config as parameter.

    Anti-patterns to avoid (from research):
    - No Popen (deadlock risk per Phase 2 decision)
    - No sys.exit() in library code
    - No load_dotenv() at import time
    - No hardcoded prompt text in Python source
  </implementation>
